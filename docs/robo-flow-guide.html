요구사항 한 문장이 소프트웨어 설계가 되기까지

Robo Architect 전체 흐름 · 일반인용 안내
Robo Architect는 사람이 평소 쓰는 말로 "이런 기능이 필요해"라고 적으면, 그것을 개발자가 실제로 쓰는 소프트웨어 설계도로 자동으로 바꿔 줍니다. 게다가 회사에 이미 있는 옛 프로그램(레거시 코드)을 인공지능(AI)이 참고해서, 현실에 맞는 설계를 만들어 줍니다. 아래는 그 전체 과정입니다.
1단계 · 입력

말로 요구사항을 적는다

개발 용어를 몰라도 됩니다. 예를 들어 “배송비를 주문 금액과 회원 등급에 따라 다르게 계산하고, 무료배송 조건을 적용한다”처럼 한 문장이면 충분합니다.

▼
2단계 · AI 분석

AI가 옛 코드를 참고하며 분석한다

AI가 자동으로 회사의 기존 코드 창고를 뒤져, 이 요구사항과 관련된 실제 코드를 찾아 참고합니다. 화면 오른쪽에 진행 과정이 실시간으로 흘러갑니다.

이때 AI는 그래프 전체를 통째로 가져오지 않고, 필요한 부분만 골라 참고합니다.
▼
3단계 · 전략 설계

업무 단위로 나뉜 설계도가 완성된다

요구사항이 ‘배송’, ‘회원등급’ 같은 업무 단위로 자동으로 나뉘고, 각 업무마다 “누가·무엇을·왜 하는지”(사용자 스토리)와 상황별 규칙(시나리오)이 그려집니다.

특히 옛 코드의 실제 규칙 — 예: ‘주문금액 5만원 이상 무료배송, 도서산간(제주 등) 3천원 추가’ — 이 설계의 근거로 그대로 인용됩니다. AI가 상상한 게 아니라 진짜 코드를 본 것입니다.
▼
4단계 · 기술 방향(헌장)

어떤 기술로 만들지 AI가 추천한다

본격 설계 전에 “하나로 묶을까(모놀리스), 잘게 나눌까(마이크로서비스)?”, “어떤 기술 스택?”을 정합니다. AI가 근거와 함께 추천을 제시하고, 사람은 확인만 하면 됩니다.

이 예시에서 AI는 “업무가 단순하니 모놀리스가 최적, 기술은 Spring Boot + PostgreSQL”을 추천했습니다.
▼
5단계 · 상세 설계(Plan)

개발자가 바로 쓸 상세 설계를 만든다

데이터 구조·명령·이벤트 등 구체적인 설계(전술 설계)가 만들어집니다. 앞서 정한 기술 방향(헌장) 위에서 결정됩니다.

▼
6단계 · 코드 생성

마지막으로 실제 코드가 만들어진다

완성된 설계를 바탕으로 실제 프로그램 코드가 자동 생성됩니다. (이번 검증은 이 코드 생성 직전 단계까지 확인했습니다.)

한 문장의 요구사항이 → 완성된 설계도가 됩니다.
사람은 방향만 확인하고, 반복 작업은 AI가 대신합니다.
