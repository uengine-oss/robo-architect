인텐트 실행 멈춤(hang) 버그 — 원인 규명과 수정 보고

Robo Architect (robo-architect) · Windows 환경 · 2026-07-11

한 줄 요약

Windows에서 Proposal의 인텐트/Plan 실행이 무한 대기(hang)에 빠져 결과가 전혀 안 나오던 문제. 원인은 AI(claude) 실행을 claude.CMD(배치 파일) 경유로 하면서 실시간 출력 통로가 끊긴 것. claude.exe를 직접 실행하도록 고쳐 해결.

1 · 증상

무엇이 안 됐나

Windows 환경에서 요구사항을 넣고 실행하면, 인텐트(요구사항 분해)가 끝나지 않고 멈춰 INTENT_FAILED만 돌려줬습니다. 설계 결과가 전혀 생성되지 않았습니다. 사장님 PC 같은 다른 환경에선 되기도 해서 더 헷갈렸습니다.

✘ AI 실시간 세션(-p) — 출력 0바이트 무한 대기 ← 범인

3 · 근본 원인

왜 멈췄나 — 비유로 이해하기

한 줄 비유: AI는 답을 수도꼭지처럼 실시간으로 조금씩 흘려보냅니다(스트리밍). 그 물(답)을 받으려면 호스가 AI에 곧바로 이어져야 하는데, Windows에선 중간에 ‘중계 밸브(cmd.exe)’가 끼면서 그 물길이 이어지지 않아, 프로그램이 물을 영영 못 받고 멈췄습니다.
✅ 정상 (AI에 직접 연결) [Robo Architect] ◀━━ 실시간 답 ━━ [AI · claude.exe] → 잘 받음 ❌ 문제 (중계 밸브 경유 = 원래 방식) [Robo Architect] ◀━ ✕ ━ [중계 cmd.exe] ◀━ 실시간 답 ━ [AI · claude.exe] ▲ └ 여기서 실시간 물길이 안 이어짐 → 프로그램은 영원히 대기
Q. 왜 하필 Windows에서?
Windows에선 claude 명령의 정체가 ‘바로가기 파일(claude.CMD)’입니다. 이 바로가기가 cmd.exe(검은 명령창)를 한 번 거쳐서 진짜 AI 프로그램(claude.exe)을 부르기 때문에, 그 중간에 물길이 끊깁니다.
Q. 왜 어떤 건 되고(버전 확인), 어떤 건 안 됐나(요구사항 분석)?
버전 확인처럼 짧게 끝나는 일은 답이 짧아 한 번에 툭 전달됩니다(물길이 필요 없음). 하지만 요구사항 분석처럼 실시간으로 길게 이어 답하는 일은 물길이 계속 연결돼 있어야 하는데, 그게 끊겨서 멈췄습니다. 그래서 “되기도 하고 안 되기도” 해 원인 찾기가 헷갈렸습니다.
Q. 왜 다른 PC(사장님 환경)에선 됐나?
그 환경은 프로그램을 켜는 방식이 달라서, 이 ‘중계 밸브(cmd.exe)’ 경로를 우연히 안 밟았기 때문으로 보입니다. 같은 코드라도 실행 방식에 따라 문제를 밟을 수도, 피할 수도 있었습니다.
실측 근거: 중계 밸브를 거치는 방식은 무한 대기(0바이트, 멈춤). AI 프로그램(claude.exe)에 직접 호스를 연결하니 1.3초 만에 정상 작동함을 확인.

4 · 수정

중계 밸브를 빼고, AI에 직접 연결

해결: 중간의 ‘중계 밸브(cmd.exe)’를 거치지 말고, AI 프로그램(claude.exe)에 호스를 곧바로 연결했습니다. 중계 밸브가 사라지니 실시간 물길이 안 끊깁니다.

기술적으로는, AI 실행 시 claude.CMD(바로가기) 대신 실제 claude.exe를 직접 실행하도록 바꿨습니다(_resolve_claude_bin()). 만약 claude.exe를 못 찾으면 예전 방식으로 되돌아가므로 더 나빠지지는 않습니다(안전장치). 추가로, 무인 자동 실행이라 표준입력(stdin)을 명시적으로 닫아 불필요한 입력 대기도 막았습니다.

5 · 검증

실물 E2E로 확인

수정 후 프론트에서 직접 실행해 끝까지 확인했습니다:

✔ 요구사항 입력 → 인텐트 분해 완주 (오류 0)
✔ AI가 레거시 코드를 실제 참조 (cluster_retrieve 5회 호출, 코드 규칙 인용)
✔ 헌장 · Plan 단계도 정상 실행

함께 반영한 부수 작업

• 레거시 코드 의미검색(robo-cluster MCP) 배선 — 인텐트 스킬이 요구사항 관련 옛 코드를 자동 참조하도록 연결(analyzer의 검색 기능을 AI가 호출).
• 프론트엔드 마커 정리 — 검색용 내부 라벨(EMBEDDED)이 화면에 역할로 잘못 노출되지 않도록 수정.
• analyzer/frontend 커밋 및 서브모듈 버전 반영.
