# MembershipManagement Bounded Context

## Overview

This bounded context implements the **MembershipManagement** domain using **Clean Architecture** for a core domain.

## Architecture Layers

### 1. Entities Layer (`entities/`)
Contains enterprise business rules and domain entities.
- `MemberAccount.ts` - Core aggregate root
- `IMemberAccountRepository.ts` - Repository interface (Dependency Inversion Principle)

### 2. Use Cases Layer (`usecases/`)
Contains application-specific business rules and command handlers.
- Command handlers will be added here when commands are defined in the design

### 3. Interface Adapters Layer (`interface_adapters/`)
Contains controllers, presenters, and adapters that convert data between use cases and external systems.
- Controllers will be added here when commands are defined

### 4. Frameworks & Drivers Layer (`frameworks_and_drivers/`)
Contains external frameworks, databases, and infrastructure concerns.
- `MemberAccountRepository.ts` - Concrete repository implementation

## Repository Usage

```typescript
import { MemberAccountRepository, IMemberAccountRepository } from './index';

// Create repository instance
const repository: IMemberAccountRepository = new MemberAccountRepository();

// Find by ID
const account = await repository.findById('account-123');

// Save
await repository.save(account);
```
