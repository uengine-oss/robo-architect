**042 Staged DDD Decomposition Mode** (started 2026-06-14) — Proposal 입력 시 **분해 모드 스위치**(Simplified vs Detailed DDD). Simplified=현행 `Intent→Plan` 무변경. Detailed=ddd-starter 6단계(**Discover→Decompose→Strategize→Connect→Define→Tactical**)를 **스테이지별 독립 스킬**(Principle X, `extends:` 각 ddd-starter reference)로, 각 단계가 그 단계 고유의 **의사결정 질문**(차별성·Core/Supporting/Generic·loose coupling·Event/Command/Query pub/sub vs sync·Aggregate 경계/invariant)을 제시하고 **확정/생략 게이트**(Principle IV)·SSE(Principle III) 로 진행. 먼저 `robo-proposal-scope`(extends 00-orientation)가 **스코프 분류 → 스테이지 플랜**(단일 BC면 cross-context 생략, 전략만이면 Tactical 생략, 마이크로면 Simplified 로 축소; Discover 완전생략 불가) 제안→아키텍트 확정. **핵심결정**: 지속 전략결정(차별성/가치제안·BC별 Core/Supporting/Generic·기본 결합 posture·BC별 ubiquitous language)은 **한 번** 기존 **Constitution 노드**(041 프로젝트루트+BC, effective 병합)의 신규 `strategicMemory` 속성에 승격 저장→후속 proposal 은 재질문 없이 confirm/amend 재사용(SC-004); 변경별 전술상세는 Proposal 에만. 충돌(메모리 vs 로컬결정)은 amend-or-justify 게이트(FR-019), 메모리 수정→의존 plan stale(기존 staleness 재사용, FR-021). **신규 노드/관계 없음** — `Proposal`(+decompositionMode/stagePlan/stageArtifacts/currentStage/memoryConflicts)·`Constitution`(+strategicMemory) 속성만. 스테이지 결과는 표준 strategicDiff/tacticalDiff 로 **수렴**→기존 plan/impact/tasks/implement 무분기(FR-023). 신규 백엔드 `proposals_staged.py`+`staged_runner`/`stage_runners/*`/`strategic_memory`/`staged_consolidate`; 프런트 `StagePlanReview.vue`/`StagedDddWalkthrough.vue`+모드 스위치. Constitution PASS(I~X). Phase 0/1 ✅, **구현 완료**(70 태스크; 단위테스트 20 신규 pass, 기존 32 무회귀; 프런트 빌드 통과; app 부팅 OK). 잔여: T068 매뉴얼 생성·T070 라이브 quickstart(런타임 백엔드+Neo4j+claude 필요). **UX 개정(2026-06-14)**: 스테이지를 별도 위저드가 아니라 **기존 탭에 융합** — 전략(Discover/Decompose/Strategize)=**Intent 탭**, 전술(Connect/Define/Tactical)=**Plan 탭**, Impact 탭 무변경; 각 단계는 `StageRunner`+단계별 시각화(Strategize 4분면 Core Domain Chart·Connect BC 컨텍스트맵·Define BCC v5·Tactical Aggregate Design Canvas v1)+객관식 컨트롤+단계별 피드백 재생성(`?feedback=`)+완료단계 클릭하여 읽기전용 재열람. 진행중 산출물은 store 초안(stageDrafts)+in-flight run 재사용으로 탭 전환에도 **재시작 안 함**. **라이프사이클 재매핑**: **DRAFT=Intent, SUBMITTED=Plan**, IMPLEMENTING~ 유지. 제출=Intent 완료(strategicDiff 만; `proceedToPlan`=전략 수렴+submit). **Plan 게이트(확정 plan+!stale)는 submit→implement 로 이동**. Detailed 의 Plan 단계는 전술 재도출 안 하고(이미 DDD 산출) **아키텍처 전용**(`architecture_only`) — 배포/ingress/mesh·프레임워크/프론트/레포매핑+연동/채널/개발환경만.

Constitution·Plan 은 `robo-proposal-tasks`(검토)·`robo-proposal-implement`(입력)까지 전달. 핵심결정(2026-06-11 개정): Constitution 은 **Neo4j 노드**(레포 파일 아님) — **프로젝트 루트 싱글톤 + BC별 오버라이드**(`(:BoundedContext)-[:HAS_CONSTITUTION]->(:Constitution)`), BC 유효헌장=루트+오버라이드 병합. 관리 UI 는 **Design 쪽**(캔버스 툴바 📜=PROJECT, 각 BC 노드 헤더 📜=BC; `robo:open-constitution` 앱이벤트+App.vue 모달, 신규 `frontend/src/features/constitution/`). **Proposals 탭엔 헌장 보기/수정 없음** — 헌장 부재 시 **인터뷰만 1회**(Plan 게이트, PlanView 인라인) → 프로젝트 루트 노드 생성, 프로포절별 헌장 절대 없음. 신규 백엔드 `api/features/constitution/`(store: 루트/BC CRUD+effective 병합). Plan 은 `Proposal.implementationPlan` JSON; staleness=plan 스냅샷 해시 vs 현재 루트 헌장 해시(`Proposal.constitutionHash`). 신규 스키마(Constitution 노드+HAS_CONSTITUTION). 스킬-우선(Principle X): `robo-proposal-plan`(extends `robo-proposal-intent`; 다수 BC 면 ddd-starter Connect 기반 연동분석 req/res vs pub/sub + Kafka 채널 + 서비스별 Docker 개발환경/멀티레포 범위), `robo-project-constitution`(extends **speckit-constitution**, 프롬프트 시드+의도적합 추천+의존성 인지 최소질문). Submit 게이트=확정 plan+`planStale=false`. Constitution PASS(I~X, 그래프-원천 강화). Phase 0/1 ✅, 구현+매뉴얼 ✅.

**039 Proposal Lifecycle Management** (started 2026-06-05) — 038 `RequirementChange(CHG-NNN)` 패러다임을 **Proposal(PRO-NNN) 기반 생애주기**로 진화. 핵심 3요소: ① AI 인텐트 분해(`robo-proposal-intent` 스킬) → Strategic Diff(Epic/Feature/UserStory) + Tactical Diff(Aggregate/Command/Event/VO). ② Git Worktree 샌드박스 — **원천은 robo-architect가 아니라 Claude Code 탭의 대상 프로젝트(`projectRoot`)**, `<projectRoot>/.sandbox/proposal/<PRO-NNN>` worktree. 구현은 헤드리스가 아니라 **Code 탭의 살아있는 Claude Code 셀(PTY) 재사용** — 중지·피드백 가능, worktree별 독립 세션을 상단 탭으로 동시 전환(멀티 세션), 백엔드 PTY 세션 레지스트리로 **새로고침에도 재어태치**(스크롤백 replay). IMPLEMENTING 이후 상태에서도 "다시 구현하기" 가능. ③ Dual Merge(코드 머지 + Neo4j 그래프 업데이트 보상 트랜잭션). 상태: `DRAFT→SUBMITTED→IMPLEMENTING→TESTING→PENDING_ACCEPTANCE→ACCEPTED/DESTROYED`(+`MERGE_FAILED`). 4개 신규 스킬(`robo-proposals/`). 038 EFFECT·SemanticDiff·SSE 재사용, 029 Claude Code 셀(`/api/claude-code/terminal`)·`openClaudeCode`/`<KeepAlive>` 재사용. Constitution PASS(I~VII+X). Phase 0/1 ✅, 인터랙티브 셀·멀티 세션·재어태치·다시 구현 구현 완료.
