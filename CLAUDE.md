Active feature plan: [specs/032-desktop-startup-picker/plan.md](specs/032-desktop-startup-picker/plan.md)

Read the plan for technologies, project structure, constitution gates, and architectural constraints relevant to current work. Companion artifacts in the same directory:
- spec.md (what & why — First user-visible screen of the Electron desktop build. Resolves session identity from `git config user.name`/`user.email` (no separate login), lists saved + Neo4j-Desktop-discovered + bundled Neo4j connections with status badges, lets user manage them (add/edit/delete), picks a project root, then hands off `(connection, project root, identity)` to the main SPA. Identity is propagated to every backend write via `X-User-Name` + `X-User-Email` headers — laying foundation for a follow-up audit/history feature. Web/server deploy unaffected. No new HTTP endpoints, no Neo4j schema change.)
- research.md (D1 same `BrowserWindow` + Vue route `/__launcher` not separate splash window, D2 read-only filesystem scan of Neo4j Desktop `Application/relate.config.json` + per-DBMS `neo4j.conf` for bolt port — silently degrades on any missing/locked/schema-mismatch, D3 `child_process.spawn("git","config","--get",...)` with `cwd=projectRoot` + 2 s timeout lets git's own precedence (env > project-local > global > system) resolve naturally, D4 in-place atomic settings v1→v2 migration — pre-existing `externalNeo4j` becomes first `SavedConnection`, keychain re-keyed `neo4j.password`→`connection.<id>.password`, D5 two plain HTTP headers no signing — trust model is local single-user tool, D6 `dialog.showOpenDialog openDirectory+dontAddToRecent`, D7 in-app Settings re-opens launcher modally; LLM keys stay in Settings.)
- data-model.md (**No Neo4j schema change, no new Pydantic request/response models.** TS types in `desktop/src/shared/launcher-contract.ts` — `SavedConnection` {id, label, uri, user, database?, source: manual|imported-from-neo4j-desktop|bundled|manual-migrated-from-023, lastConnectedAt, createdAt}, `DiscoveredConnection` {discoveryId, dbmsName, uri, neo4jVersion, status, projectName} transient, `ProjectRootEntry` {path, lastUsedAt} max 5, `LaunchProfile` {connectionId, projectRoot, enteredAt}, `SessionUser` {name, email, source: env|project-local-git|global-git|system-git|unknown-fallback, displayName}. `DesktopSettings` v2 adds `savedConnections[]`, `recentProjectRoots[]`, `lastProfile`. Keychain key `connection.<id>.password`. One Python `Actor` dataclass on `request.state.actor`.)
- contracts/launcher-ipc-contract.md (Extends `window.desktop` — adds `connections:{list,save,update,delete,discoverNeo4jDesktop,probeStatus,test}`, `projectRoot:{choose,listRecent,validate}`, `identity:resolve`, `launcher:{enter,reopen}`. Reuses 023's `IpcResult<T>` envelope; extends closed `IpcErrorCode` with `CONNECTION_DUPLICATE/NOT_FOUND`, `DISCOVERY_UNAVAILABLE`, `PROJECT_ROOT_INVALID/UNREADABLE`, `GIT_UNAVAILABLE`, `LAUNCHER_ALREADY_ENTERED`. Discovery never errors — empty list on any failure.)
- contracts/identity-header-contract.md (`X-User-Name` UTF-8 percent-encoded + `X-User-Email` ASCII. Frontend interceptor in `frontend/src/app/http.ts` reads `useSession().user` per-request. Backend Starlette middleware `api/platform/identity/middleware.py` registered after correlation-ID middleware reads headers → `request.state.actor: Actor(name,email,source)`. Missing/empty headers fall back to `unknown@<hostname>` — never 401s. Trust model: local-loopback only; not safe for public-facing deploy.)
- quickstart.md (8 manual smoke scenarios across Windows + macOS — Q1 cold first-time setup, Q2 returning-user one-click resume <5s, Q3 git identity precedence env>local>global, Q4 unknown-user fallback (no git), Q5 Neo4j Desktop discovery + negative test on hosts without it, Q6 manage saved connections + keychain cleanup, Q7 web-mode unaffected, Q8 v1→v2 migration with externalNeo4j auto-import. Out-of-band: password-leak grep, cold-start budget ≤1.5s, identity-propagation audit.)
