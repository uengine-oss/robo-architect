Read the plan for technologies, project structure, constitution gates, and architectural constraints relevant to current work. Companion artifacts in the same directory:
- spec.md (what & why — Every LLM-generated natural-language artifact in the product — user stories, acceptance criteria, ingestion-workflow event/aggregate/command/UI-wireframe descriptions, change-management plan narratives, DDD spec prose, requirements clarification — must be produced in a language the user selects from the gear-icon Settings panel. Default = `navigator.language` (BCP-47). UI i18n out of scope. Existing stored artifacts not retranslated. - research.md (D1 `Accept-Language` standard header set globally by the SPA HTTP interceptor — no custom header / body field / query param; D2 server-side fallback = **`en-US`** when no header, overridable via `GENERATION_LANGUAGE_DEFAULT` env var (resolves FR-010); D3 single shared `build_system_message()` in `api/platform/llm_messages.py` appends "Respond in {tag}" directive — every existing `SystemMessage(content=...)` call site refactored; D4 per-request `ContextVar` in `api/platform/language.py` mirroring the existing `_request_id_var` pattern; D5 AST-based pytest regression in `api/tests/regression/test_language_chokepoint.py` fails if any `api/features/` file uses `SystemMessage(...)` directly; D6 all 74 audited call sites refactored in one PR; D7 Pinia store at `frontend/src/app/language.store.js` mirrors `theme.store.js` pattern; D8 user-supplied input text preserved verbatim — language policy applies only to LLM-generated prose.)
- data-model.md (**No Neo4j schema change, no new Pydantic models, no new persisted server-side schema.** §Effective Language (client-side) — BCP-47 tag in `localStorage.app_language` + Pinia `useLanguageStore`, default lazy from `navigator.language`. §Language Context (server-side, per-request) — `ContextVar[str|None]` in `api/platform/language.py` set by middleware reading `Accept-Language`; fallback to `GENERATION_LANGUAGE_DEFAULT` env (`en-US`). §Generated Natural-Language Output — every `SystemMessage` in `api/features/` constructed via `build_system_message(content)` chokepoint; AST regression test enforces no direct `SystemMessage(content=...)` outside `api/platform/llm_messages.py`.)
- contracts/language-policy-contract.md (Three contracts. C1: HTTP `Accept-Language` header — single BCP-47 tag, normalized server-side (split-on-comma + strip-q-values + case-normalize); header absence = env default. C2: backend chokepoint — `api/platform/llm_messages.build_system_message(content: str) -> SystemMessage`, appends language directive, single-positional-arg, test-only `_skip_language_directive` escape. C3: frontend chokepoint — `useLanguageStore` + Axios request interceptor at `frontend/src/services/httpClient.js`; Settings UI adds Language section to [SettingsPanel.vue](frontend/src/app/layout/SettingsPanel.vue) with `<datalist>` of recommended tags but free-form input allowed. Explicit non-contracts: no new REST endpoints, no new response headers, no UI i18n, no retroactive translation.)
- quickstart.md (9 manual smoke scenarios — S1 default = browser locale per US1, S2 explicit Settings override persists across reload+restart per US2, S3 stored artifacts untouched per SC-007, S4 chokepoint inheritance + AST regression deliberate-failure check per US3, S5 no-header request hits env default per FR-010, S6 exotic BCP-47 tag pass-through per FR-011, S7 graceful degradation when `localStorage` blocked, S8 orthogonal composition with Domain Terminology toggle per FR-012, S9 non-regression vs FR-013.)
